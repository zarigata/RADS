### WebSocket Server

```rads
blast main() {
    turbo server = websocket_new(8080);

    websocket_on_message(server, handle_message);
    websocket_on_connect(server, handle_connect);

    websocket_listen(server);
}

blast handle_message(client, msg) {
    echo("Received: " + msg);
    websocket_send(client, "Echo: " + msg);
}

blast handle_connect(client) {
    websocket_send(client, "Welcome!");
}
```

**Test:**
```bash
# Terminal 1
$ rads websocket_echo.rads
Server listening on 8080

# Terminal 2
$ websocat ws://localhost:8080
Connected
Hello
Echo: Hello
```

### GraphQL Server

```rads
blast main() {
    turbo schema = graphql_schema();

    // Define type
    graphql_add_type(schema, "User", ["id: ID!", "name: String!"]);

    // Define query
    graphql_add_query(schema, "user(id: ID!): User", get_user);

    // Start server
    graphql_server(8080, schema);
}

blast get_user(args) {
    turbo user = {
        "id": args.id,
        "name": "John Doe"
    };
    return user;
}
```

**Query:**
```graphql
query {
  user(id: "1") {
    id
    name
  }
}
```

**Response:**
```json
{
  "data": {
    "user": {
      "id": "1",
      "name": "John Doe"
    }
  }
}
```

## üèóÔ∏è Architecture Highlights

### Modularity
- **Bytecode VM** - Separate compilation target from interpreter
- **Profiler** - Independent module with clean API
- **WebSocket/GraphQL** - stdlib extensions, not core
- **LSP Server** - Standalone Rust binary
- **Debug Protocol** - Pluggable system

### Clean Separation
- Core runtime unchanged
- Bytecode execution as alternative path
- All new features backward compatible
- Legacy interpreter available via `--interpreter` flag

