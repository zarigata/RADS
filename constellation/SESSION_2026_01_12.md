### 3. Design Decisions

**Architecture: "Constellation"**
- STARs = Individual RADS instances
- CONSTELLATION = Cluster of STARs
- CONTROLLER = Orchestrator daemon

**Key Technologies:**
- Linux namespaces/cgroups (isolation)
- Gossip protocol (node discovery)
- Raft consensus (leader election)
- DHT (distributed hash table)
- MessagePack (serialization)
- TLS 1.3 (encryption)

**Performance Targets:**
- 100ms instance startup (vs 10+ sec for containers)
- 10MB per instance (vs 100+ MB for containers)
- < 1ms network latency (same datacenter)
- 1000+ node clusters
