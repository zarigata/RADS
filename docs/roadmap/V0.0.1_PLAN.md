# RADS v0.0.1 - Release Plan

## Vision
Create a revolutionary retro-styled programming language with modern tooling that developers worldwide can install and use in minutes.

## Target Release Date
2-3 weeks from now

## Killer Features (The Big 3)

### 1. üéÆ Interactive REPL
**Goal:** Learn RADS in 5 minutes, test code instantly

**Features:**
- Start with just `rads` (no arguments)
- Multi-line input support
- Variable persistence across lines
- Auto-completion for stdlib functions
- Syntax highlighting in terminal
- History with arrow keys
- `.help` `.exit` `.clear` commands

**Implementation:**
- Add REPL mode to src/core/main.c
- Use linenoise or readline for input
- Incremental statement execution
- Environment persistence

### 2. üí° Excellent Error Messages
**Goal:** Make debugging a joy, not a pain

**Features:**
- Show source code context (3 lines before/after error)
- Helpful suggestions ("Did you mean 'echo' instead of 'eco'?")
- Stack traces with function names
- Color-coded errors (red for errors, yellow for warnings)
- Common fix suggestions

**Examples:**
```
Error: Undefined variable 'messge' at line 10
  8 |   turbo message = "Hello";
  9 |   echo("Starting...");
 10 |   echo(messge);
     |        ^^^^^^
     |        Did you mean 'message'?
```

**Implementation:**
- Enhance error_at() in parser.c
- Add source code tracking
- Build suggestion engine
- Add color support with ANSI codes

### 3. üì¶ Working Package Manager (RADStar)
**Goal:** First retro language with modern package management

**Features:**
- `rstar install <package>` - Install from registry
- `rstar search <term>` - Search packages
- `rstar list` - List installed packages
- `rstar remove <package>` - Uninstall
- `package.rads` manifest format
- Git-based package registry

**Implementation:**
- Complete tools/rstar/rstar.c
- Add HTTP download support (libcurl)
- Create package format parser
- Build local registry
- Create initial package index

### 4. üåê Web Framework (RadsWeb)
**Goal:** Build production web apps in RADS

**Features:**
- Middleware system
- Enhanced routing with parameters
- Request/Response objects
- Template engine (simple {{ var }} syntax)
- Session management
- Cookie support
- Form parsing
- File uploads

**Implementation:**
- Extend stdlib_net.c
- Add middleware chain
- Create template parser
- Add session storage

### 5. üì• Easy Installation
**Goal:** Install RADS in one command anywhere

**Features:**
- Linux installer: `curl -sSL rads.dev/install.sh | sh`
- Pre-built binaries for:
  - Linux x86_64
  - macOS x86_64
  - macOS ARM64 (M1/M2)
  - Windows x86_64
- Docker image: `docker run rads/rads`
- Package managers:
  - Homebrew (macOS)
  - APT/YUM (Linux)

**Implementation:**
- Create release build script
- Cross-compile for all platforms
- Build Docker image
- Write install.sh script
- Create package specs

**Risk:** Cross-platform builds fail
**Mitigation:** Start with Linux, add others incrementally

**Risk:** Package manager too complex
**Mitigation:** Start with simple Git-based system

**Risk:** Features take too long
