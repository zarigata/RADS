### 4. **String Concatenation** (Enhanced)
Automatic type conversion for all value types.

**Syntax:**

**Supported Types:**

- String + Integer

### 5. **Array Methods** (NEW!)
Essential array operations.

**Syntax:**
```rads
nums = [1, 2, 3];
echo(nums.length);  // 3

nums.push(4);       // Adds 4 to end
last = nums.pop();  // Removes and returns 4
```

**Methods:**
- `.length` - Property returning array size
- `.push(value)` - Add element to end
- `.pop()` - Remove and return last element

---

### 6. **Import System** (NEW!)
Basic file-based module system.

**Syntax:**
```rads
// lib.rads
struct Point { i32 x; i32 y; }
blast distance(p1, p2) { ... }

// main.rads
import "lib.rads";

p = Point { x: 10, y: 20 };
```

**Features:**
- File-based imports with string literals
- Imports structs successfully
- Imports enums successfully
- Imports functions successfully
- 3-pass execution system

**Limitations:**
- Requires full file paths (no module resolution yet)

### 7. **Enhanced Math Module** (Expanded)
18 mathematical functions.

**New Functions:**
- `math.log(x)` - Natural logarithm
- `math.log10(x)` - Base-10 logarithm
- `math.exp(x)` - Exponential (e^x)
- `math.clamp(value, min, max)` - Constrain value to range
- `math.lerp(a, b, t)` - Linear interpolation

**Existing Functions:**
- `math.abs(x)` - Absolute value
- `math.sqrt(x)` - Square root
- `math.pow(base, exp)` - Power
- `math.floor(x)`, `math.ceil(x)`, `math.round(x)` - Rounding
- `math.sin(x)`, `math.cos(x)`, `math.tan(x)` - Trigonometry
- `math.min(a, b)`, `math.max(a, b)` - Min/max
- `math.random()` - Random float [0, 1)
- `math.random_int(max)` - Random integer

**Example:**
```rads
echo(math.sqrt(16));           // 4
echo(math.pow(2, 8));          // 256
echo(math.clamp(15, 0, 10));   // 10
echo(math.lerp(0, 100, 0.5));  // 50
```

## üí° Known Limitations

### Parser Limitations:
- Negative number literals in expressions not supported
  - **Workaround:** Use `x = 0 - 42` instead of `x = -42`

- If/else statements in function bodies cause parsing errors
  - **Status:** Separate parsing issue, will be fixed in v0.0.3

### Function Limitations:
- All parameters are passed by value (no references)
- No default parameter values
- No variadic functions

### Module System:
- No module resolution (requires full paths)
- No circular import detection
- No module caching
