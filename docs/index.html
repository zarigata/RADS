RADS Programming Language

A lightweight, low-level language built for the early-internet spirit: fast to build, async by default, and ready to run on tiny machines, microservers, and always-on tools.

Copy the 1-line install View on GitHub

Install in One Line

Run this in your terminal to install RADS (uses /usr/local by default; set USE_SUDO=no and RADS_PREFIX=$HOME/.local for user installs):

curl -fsSL https://raw.githubusercontent.com/zarigata/rads/main/install.sh | bash

After install, run rads --help. Ensure $PREFIX/bin is on your PATH.

Why RADS

Turbo-speed, low-level

Near-C performance with async-first primitives. Designed to live close to the metal yet stay simple.

Microserver-ready

Perfect for tiny VPS, IoT boards, and retro hardware. Minimal footprint, fast startup, dependable runtimes.

Async baked in

Built to stream, serve APIs, and react to events with native async keywords and network-oriented stdlib.

2000s-internet spirit

Bold, direct, and practical. RADS keeps the no-nonsense ethos of early web builders while embracing modern safety.

Roadmap

Short-term: richer HTTP stack, JSON helpers, directory utilities, better error handling, bytecode VM prototype.

Mid-term: package registry, REPL, debugger hooks, editor extensions, WebSocket support.

Long-term: game-focused modules, graphics/audio helpers, and a compact VM with JIT experiments.
